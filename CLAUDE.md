# spec-canvas — 화면 생성 규칙

이 저장소는 오픈소스 템플릿이다. 만든 사람이 모르는 다른 사용자들도 이 템플릿으로
자신의 화면기획을 만든다. 아래 규칙을 반드시 지킨다.

## 규칙

1. **화면기획을 작성/수정할 때는 `docs/` 폴더의 문서만 읽는다.** 이 저장소 밖의
   다른 프로젝트, 과거 작업 이력, 특정 회사·도메인 지식은 참조하지 않는다.
   - 이유: 이 템플릿은 만든 사람이 모르는 다른 사용자들도 쓰기 때문에, 특정
     회사·도메인 지식이 섞여 들어가면 안 된다.
2. **사용자가 화면 생성 시 특정 방향(레이아웃, 우선순위, 특정 컴포넌트 사용 등)을
   직접 지시하면, 그 지시가 `docs/` 문서 해석보다 우선한다.**
3. 화면은 `components/`에 있는 기존 아토믹 컴포넌트만 조합해서 구성한다. 필요한
   컴포넌트가 없으면 새로 만들지 않고 사용자에게 먼저 확인한다.
4. `index.html`의 좌측 라우트 목록과 우측 spec은 한국어/영어 병기로 작성한다.
   - 좌측 인덱스: `ROUTES` 배열(그룹 → items)에 라우트 추가.
   - 우측 spec: `assets/js/specs.js`(한국어) / `assets/js/specs-en.js`(영어)에
     같은 키로 항목 추가.
   - 중앙 화면 자체를 한/영으로 보여주려면 `components/en/<파일명>.html`에 영문
     버전을 만든다 — 상단 🌐 전역 토글이 이 경로를 자동으로 로드한다.
5. 이 저장소 자체(엔진, 컴포넌트, CSS)에는 특정 회사·프로젝트의 실제 도메인
   콘텐츠(상품명, 실제 고객사명, 실제 정책 수치 등)를 절대 추가하지 않는다.
   샘플이 필요하면 제네릭 placeholder를 쓴다.

## 구조

```
spec-canvas/
  index.html              # 3분할 뷰어 엔진
  components/             # 아토믹 컴포넌트 + 조합 예시 페이지(x-*.html)
  components/en/          # 위 파일들의 영문 버전 (선택)
  assets/css/              # tokens.css / admin-real.css / admin-components.css / harness-chrome.css
  assets/js/                # specs.js / specs-en.js / admin-shell.js
  docs/                    # 여기에 본인의 기획서·정책 문서를 넣는다
  docs/plans/               # (선택) 본인의 구현 계획 문서
  docs/specs/                # (선택) 본인의 화면/정책 설계 문서
```
