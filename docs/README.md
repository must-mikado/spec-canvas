# docs/

이 폴더에 본인의 기획서·정책 문서를 넣는다 (마크다운, PDF, 텍스트 등 형식 무관).

넣은 뒤 Claude Code에게 이렇게 요청한다:

> "docs 폴더의 문서 보고 화면 그려줘"

Claude는 `docs/`에 있는 문서만 근거로 삼아 `components/`의 기존 아토믹
컴포넌트를 조합해 `index.html`의 라우트(좌측 인덱스)와 한/영 spec(우측
description)을 작성한다. 화면 생성 방향에 대해 원하는 지시(레이아웃, 우선순위,
특정 컴포넌트 사용 등)가 있으면 요청 시 함께 알려주면 그 지시가 우선 반영된다.

이 폴더는 처음에는 비어 있다. 여기 넣는 문서 외에는 참조하지 않는다는 것이
`CLAUDE.md`의 규칙이다.

`docs/plans/`, `docs/specs/`는 본인의 기획·정책 문서를 넣을 때 쓰라고 미리
만들어둔 하위 폴더다 — 구현 계획 문서는 `docs/plans/`에, 화면/정책 설계 문서는
`docs/specs/`에 넣는 식으로 정리하면 된다. 필수는 아니고, `docs/` 바로 아래에
넣어도 똑같이 읽는다.
